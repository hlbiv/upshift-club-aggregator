## 1.5. Apply the events_source_enum split (one-shot, run BEFORE push)

Splits the previously-shared `events_source_enum` by domain:
- Adds `totalglobalsports` and `usclub_sanctioned` to `events_source_enum` (reverses the prior downcasts to `'other'`).
- Creates `roster_source_enum` for `club_roster_snapshots.source` (12 values incl. CMS vendors from `cms_detect.py`).
- Backfills `events.source='other'` rows that were really TGS or US Club.

Hand-rolled because `ALTER TYPE ... ADD VALUE` cannot be referenced inside the same transaction it's added — drizzle-kit push wraps DDL in a single transaction and would fail.

**Pre-flight** (verify backfill assumptions):

```bash
psql "$DATABASE_URL" -c "SELECT DISTINCT source FROM club_roster_snapshots;"
# Expect: only NULL (the writer omits the column today).

psql "$DATABASE_URL" -c "SELECT source, count(*) FROM events GROUP BY source;"
# Note the 'other' count — should shrink after backfill.

psql "$DATABASE_URL" -c "SELECT DISTINCT platform_event_id FROM events WHERE source='other' LIMIT 50;"
# Expect mostly tgs-* and usclub-* sentinels.
```

**Apply** (idempotent — every statement uses `IF NOT EXISTS` / `ADD VALUE IF NOT EXISTS`):

```bash
psql "$DATABASE_URL" -f scripts/src/migrations/0002_split_events_source_enum.sql
```

**Verify**:

```bash
psql "$DATABASE_URL" -c "\dT+ events_source_enum"
# Expect: gotsport, sincsports, totalglobalsports, usclub_sanctioned, manual, other

psql "$DATABASE_URL" -c "\dT+ roster_source_enum"
# Expect: gotsport, sincsports, soccerwire, sportsengine, squarespace, duda,
#         360player, wordpress, wix, static_html, manual, other

psql "$DATABASE_URL" -c "SELECT source, count(*) FROM events GROUP BY source;"
# Expect nonzero counts for totalglobalsports and usclub_sanctioned post-backfill.

psql "$DATABASE_URL" -c "\d club_roster_snapshots"
# The `source` column should now show type `roster_source_enum`.
```

After this completes, Section 2's `pnpm db push` becomes a no-op for the enum types (already applied) and only picks up the 5 Wave-2 tables.

## 4. Smoke-test the API

With `UPSHIFT_DATA_API_KEY` exported:

```bash
# Docs site
curl -sSI http://localhost:8080/api/docs | head -1
# Expect: HTTP/1.1 200 OK

# Auth gate
curl -sS -o /dev/null -w "%{http_code}\n" http://localhost:8080/api/clubs?page_size=1
# Expect: 401
curl -sS -o /dev/null -w "%{http_code}\n" -H "X-API-Key: $UPSHIFT_DATA_API_KEY" \
  http://localhost:8080/api/clubs?page_size=1
# Expect: 200

# Rate limit (burst 101 requests)
for i in $(seq 1 101); do
  curl -sS -o /dev/null -w "%{http_code}\n" \
    -H "X-API-Key: $UPSHIFT_DATA_API_KEY" \
    http://localhost:8080/api/clubs?page_size=1
done | sort | uniq -c
# Expect: 100 × 200, 1 × 429

# New event-platform source filter (post-enum-split)
curl -sS -H "X-API-Key: $UPSHIFT_DATA_API_KEY" \
  "http://localhost:8080/api/events/list?source=totalglobalsports&page_size=5" | jq '.total'
# Expect: nonzero (the 0002 backfill reclassified TGS rows from 'other').
```

## 10. Rollback switches

If anything misbehaves, flip one of these in Secrets and restart the api-server / scraper:

| Flag | Off behavior |
|---|---|
| `API_KEY_AUTH_ENABLED=false` | Disables auth gate (pre-#62 behavior). |
| `API_RATE_LIMIT_ENABLED=false` | Disables rate limiting. |
| `API_DOCS_ENABLED=false` | Hides `/api/docs`. |
| `ARCHIVE_RAW_HTML_ENABLED=false` | Scraper stops archiving; DB and bucket untouched. |

Schema changes are non-destructive: the 5 new tables are empty on creation and unused by existing routes; `triggered_by` defaults to `'manual'` so existing logger calls are unchanged.

**No rollback flag for the enum split.** `ALTER TYPE ... ADD VALUE` is permanent in Postgres (no `DROP VALUE`), and the column-type swap is wrapped in its own transaction — partial failure rolls back automatically. Forward-only.
